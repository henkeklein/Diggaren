Diggaren API-dokumentation

Inledning

I den här dokumentationen kan du läsa om hur du kan använda Diggaren’s API. Det kommer finnas exempel på anrop som kan göras och vilken information du kan hämta.

Syfte

Syftet med API’t är att användaren ska kunna hämta data från olika radiokanaler hos Sveriges Radio. Sedan kan en Spotify URL skapas av den låten som spelas just nu. Vi tror att användaren i slutändan kommer använda sig av datan i syfte att utveckla en tjänst som kan kolla upp vilken låt som spelas på en viss radiokanal och skapa Spotify-länkar av låten.

Målgrupp

De som helst kommer dra nytta av API’t är musikintresserade utvecklare, som vill skapa sin egen tjänst som erbjuder musik.

Utvecklare

De som främst kommer använda API’t är vi utvecklare, som anpassar API’t för vår egna webbtjänst. Dock är API’t enkelt designat, så det inte ska vara några problem för externa utvecklare. Datan som returneras från varje URI är i JSON-format. XML är inte tillgänligt. De applikationer som kan tillverkas av vårt API är någon form av musikapplikation, som tillhandahåller information om vilka låtar som spelas på olika radiostationer. Man kan också tillverka en tjänst som låter en användare spara ner alla låtar i spellistor på Spotify. Möjligheterna är många.

Användningsområden

Att använda vårt API är väldigt enkelt. Det krävs enbart att man skriver in en korrekt URL och data i JSON-format returneras. Det finns inget stöd för XML-format. Exempel på anrop: https://diggaren.herokuapp.com/dingata

Anropet returnerar JSON-data som innehåller information om vilken låt som spelas på radiokanalen dingata. Anropet ger också en Spotify-URL till låten som spelas. De radiokanaler som är tillgängliga idag är: P3 Dingata P3 Knatte P3 Star

Anrop

Att göra ett anrop i vårt API är väldigt enkelt. Vår tjänst körs genom tjänsten Heroku, som förser oss med en egen domän. För att göra ett anrop anger du https://diggaren.herokuapp.com/{kanal}.

Tillgängliga parametrar är:

* dingata - för P3 Dingata

* star - för P3 Star

* knatte - för P3 Knatte

Utan parameter skrivs ett 404-fel ut.

Exempel på hur ett svar kan se ut:

{ "description": "David Guetta & Nicki Minaj & Afrojack - Hey Mama", "url": "https://open.spotify.com/track/5Ok52kEHxudhX1n2RvqyIe" }
